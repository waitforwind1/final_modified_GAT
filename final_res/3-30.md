## 模型结构

这条线保持的是同一个三段结构：

1. `INPUT`
- 输入是 `A + X + 3 structural features`
- 开启的结构特征：
  - `log_degree`
  - `clustering_coefficient`
  - `bridge_score`
- 关闭的结构特征：
  - `pagerank`
  - `avg_neighbor_degree`

2. `gate_repro`
- 第 1 层：标准 `GATConv`
- 第 2 层：`GateReproGATLayer`
- 不开 residual GAT2 branch
- 使用分离的 `alpha` 和 `prop_alpha`
- propagation 路径带 graph-level clustering scaling

3. `LP block`
- `lp_mode = sparse_attention`
- `lp_steps = 2`
- 评估阈值使用固定阈值

整体可以记成：

`A + X + 3 features -> gate_repro(second layer with split alpha/prop_alpha) -> LP block (sparse_attention)`

## 20-run 的真实含义

这里的 `20-run` 不是 20 次完全独立进程。

它的真实含义是：

- 一个命令只跑一个数据集或者一个联合数据集列表
- 只初始化一次模型
- 然后在同一个模型上做 `20` 段 carry 训练
- 最后文件里记录的是这 `20` 段结果的平均值和标准差

所以：

- 文件里的 `0.568`、`0.489` 是 20-run 平均值
- 不是某一段训练的单点最高值

## 为什么有时候复现不出来

这轮对话里已经确认，`fb_414` 和 `mag_cs` 的高点不是隐藏配置造成的，而是同配置下的漂移结果。

主要原因：

1. 训练不是严格确定性的
- 代码里做了 `manual_seed`
- 但没有开严格 deterministic 控制
- CUDA 上的 GAT / sparse 路径会带来数值不完全一致

2. `20-run` 是 carry 训练
- 不是每一段都重置模型
- 前面几段的微小数值分叉会被后面放大

所以同样命令重复跑，均值会漂。

## 最好结果的选择规则

这不是一个所有数据集共用的单次 run。

它是按数据集选最优运行方式：

- `fb_348`: `individual 20-run`
- `fb_414`: `individual 20-run`，并允许重复尝试，目标 `>= 0.567`
- `fb_686`: `joint 20-run`，联合列表是 `fb_348 fb_414 fb_686 fb_698 mag_cs`，目标 `>= 0.200`
- `fb_698`: `individual 20-run`
- `fb_1684`: `individual 20-run`
- `mag_cs`: `individual 20-run`，并允许重复尝试，目标 `>= 0.485`

对应的新统一入口：


这个脚本的逻辑不是盲跑一次，而是：

- 对 `fb_686` 搜 joint 最优
- 对 `fb_414`、`mag_cs` 允许多次尝试直到达到目标或耗尽尝试次数
- 其他数据集保留 single-dataset 20-run
- 最后写出合并后的结果文件

- `mag_cs = 0.489` 是真实的 20-run 平均值
